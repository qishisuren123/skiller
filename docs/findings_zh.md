## 3. 方法论

### 3.1 Skill 生成管线

1. **对话模拟**：Sonnet 模拟 12-16 轮科学家-AI 对话，包含 5+ 次 error→fix 迭代
2. **Skill 提取**：从对话中提取结构化 skill 包（SKILL.md + scripts/ + references/ + assets/）
3. **质量评估**：24 分 rubric 自动评分，50 个 skill 全部达到 23/24

### 3.2 验证实验

- **L0 条件**：不提供任何 skill，模型仅凭自身能力完成任务
- **L4 条件**：提供完整 skill 包（SKILL.md + 参考代码 + 工作流 + 踩坑指南 + 输出示例）
- **评估方式**：自动化测试脚本，输出 PASS/FAIL 判定

### 3.3 24 分 Skill 评分体系

| 组件 | 满分 | 检查项 |

## 5. 结论

1. **Skill 对弱模型的提升最为显著**（GPT-4o-mini: +15%, d=0.41），强模型几乎不受影响（GPT-4o: +0.1%）
2. **领域适配性差异巨大**：环境科学 +67% vs 化学 -23%，表明 skill 需要精准匹配领域
3. **Skill 的主要作用是帮助代码格式化**而非直接解决逻辑问题
4. **"弱模型 + skill" 可以达到甚至超过 "强模型无 skill" 的效果**，为低成本部署提供了可能
